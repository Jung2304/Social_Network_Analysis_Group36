# LightGCN on MovieLens 100K with RecBole

This project runs the LightGCN recommender model using RecBole on the MovieLens 100K dataset.
The training entry point in this repository is:

- `experiment/models/LightGCN/run_lightgcn.py`

---

## 1. Output File Structure Explanation

After running LightGCN, the working directory `experiment/models/LightGCN/` typically looks like this:

```text
LightGCN/
├─ log/
│  └─ LightGCN/
│     └─ LightGCN-ml-100k-<timestamp>-<id>.log
├─ log_tensorboard/
│  └─ LightGCN-ml-100k-<timestamp>-<id>/
│     ├─ events.out.tfevents.*
│     └─ <subrun>/
│        └─ events.out.tfevents.*
├─ saved/
│  └─ LightGCN-<timestamp>.pth
└─ run_lightgcn.py
```

### 1.1 `log/` folder

#### Purpose
Stores plain-text training logs generated by RecBole for each run.

#### File type
- `.log` files, for example:
  - `LightGCN-ml-100k-Apr-08-2026_09-58-09-79e0ff.log`

#### Information stored
- Full run configuration (general, training, evaluation, dataset hyperparameters)
- Dataset statistics (user/item count, interactions, sparsity)
- Model summary (architecture and trainable parameter count)
- Epoch-by-epoch outputs:
  - training loss
  - validation score
  - ranking metrics (Recall@K, MRR@K, NDCG@K, Hit@K, Precision@K)
- Checkpoint save events and early-stopping progress
- Final evaluation information

### 1.2 `log_tensorboard/` folder

#### Purpose
Stores TensorBoard event files for visual monitoring of training and evaluation.

#### File type
- `events.out.tfevents.*` files
- Optional subrun folders created by the logger/runtime

#### Information stored
- Scalar summaries over steps/epochs (for example, training loss and validation metrics)
- Time-series data used by TensorBoard dashboards

#### Practical usage
- **Visualization**: inspect metric curves instead of reading raw logs line by line.
- **Convergence checks**: detect overfitting or metric plateaus.
- **Run comparison**: compare multiple LightGCN runs in one TensorBoard session.

Example command:

```bash
tensorboard --logdir experiment/models/LightGCN/log_tensorboard
```

### 1.3 `saved/` folder

#### Purpose
Stores trained model checkpoints.

#### File type
- `.pth` checkpoint files, for example:
  - `LightGCN-Apr-08-2026_09-58-10.pth`

#### Information stored
A RecBole/PyTorch checkpoint typically contains:
- learned model weights (`state_dict`)
- optimizer/training state (when available)
- best validation score and related metadata
- configuration snapshot needed to reload the model consistently

#### Practical usage
- **Inference**: load a trained model without retraining.
- **Evaluation reuse**: re-run test/evaluation from the best checkpoint.
- **Training continuation**: resume from a previous state if needed.

### 1.4 `run_lightgcn_experiments.py`

#### Purpose
Minimal launcher script that calls RecBole quick start API:
- model: `LightGCN`
- dataset: `ml-100k`

#### Practical usage
- Single entry point for running baseline experiments.
- Easy to extend later with custom config files or command-line overrides.

## 2. How to Access LightGCN Source Code in RecBole

Use this section when you want to inspect or debug the actual LightGCN implementation inside the installed RecBole package.

### 2.1 Locate RecBole in your local Python environment

Your environment example:

```text
D:\Miniconda\envs\recbole\Lib\site-packages\recbole
```

This is the package root directory for RecBole in the `recbole` Conda environment.

### 2.2 Open the RecBole folder in VS Code

1. In VS Code, select **File > Open Folder...**.
2. Choose:
   `D:\Miniconda\envs\recbole\Lib\site-packages\recbole`
3. Open the folder.

### 2.3 Practical tips when inspecting source

- Use VS Code global search to find methods such as:
  - `forward`
  - `calculate_loss`
  - `full_sort_predict`
- Keep your project script (`run_lightgcn.py`) and source file (`lightgcn.py`) open side by side to map API calls to implementation behavior.
- If you only need to inspect behavior, prefer reading source first before modifying package files in `site-packages`.

## 3. Quick Start (Run + TensorBoard)

Use the following commands to run LightGCN on MovieLens 100K and visualize metrics.

### 3.1 Run training

From the project root:

```powershell
conda activate recbole
cd "W:\Ki 2_Năm 3\SNA_final-project_group36\experiment\models\LightGCN"
python run_lightgcn.py
```

What happens after running:
- A `.log` file is created in `log/LightGCN/`
- TensorBoard event files are created in `log_tensorboard/`
- The best model checkpoint is saved as `.pth` in `saved/`

### 3.2 Start TensorBoard

From the same `LightGCN` directory:

```powershell
tensorboard --logdir log_tensorboard --port 6006
```

Then open:

```text
http://localhost:6006
```
