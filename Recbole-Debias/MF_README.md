## 1. Core Idea & Architecture
### Concept
The Matrix Factorization (MF) baseline implements a standard **Latent Factor Approach**. It assumes that both users and items can be characterized by a set of latent factors (embeddings) in a low-dimensional vector space.

### Goal
The primary objective is to decompose the sparse User-Item interaction matrix $R$ into two lower-rank matrices: $P$ (User latent factors) and $Q$ (Item latent factors). By learning these embeddings, the model can predict the missing entries in the interaction matrix, representing unobserved user preferences.

### Input/Output
- **Input:** User IDs and Item IDs from the interaction history.
- **Output:** A scalar preference score $\hat{r}_{ui}$, calculated as the dot product of the corresponding user and item embeddings:
  $$\hat{r}_{ui} = p_u^\top q_i = \sum_{k=1}^d p_{uk} \cdot q_{ik}$$
  where $d$ is the dimensionality of the latent space (`embedding_size`).

## 2. Data Processing Pipeline
### Dataset
We utilize the **ml-100k** atomic files. The pipeline follows these structured steps:
1.  **Loading:** Atomic interaction files are loaded into the RecBole environment.
2.  **Filtering:** A minimal interaction filter (`min_user_inter=1`, `min_item_inter=1`) ensures data density for latent factor estimation.
3.  **Splitting:** The dataset is split into **Normal** training/validation sets and a specialized **Intervened Test Set**.
4.  **Intervene Mask:** The `intervene_mask` plays a critical role in evaluating the MF baseline under **unbiased conditions**. By filtering test interactions where popularity bias is minimized (simulating a uniform distribution), we can measure the model's true semantic precision versus its tendency to follow popularity trends.

## 5. Loss Function: BPR Loss
The model is optimized using **Bayesian Personalized Ranking (BPR) Loss**, defined as:
$$\mathcal{L}_{BPR} = -\sum_{(u, i, j) \in D} \ln \sigma(\hat{r}_{ui} - \hat{r}_{uj}) + \lambda \|\Theta\|^2$$
where $i$ is a positive interaction, $j$ is a sampled negative interaction, and $\Theta$ represents model parameters.

**Why Pairwise?** Pairwise ranking (BPR) is superior to Pointwise regression for recommendation because it focuses on the **relative order** of items rather than absolute rating values, which aligns better with the ranking-centric nature of user preference.
